# Security Policy

## Supported Versions

| Version | Supported          |
|---------|--------------------|
| 1.0.x   | :white_check_mark: |

## Reporting a Vulnerability

This is a documentation/markdown skill — no executable code is shipped at runtime. However, if you spot:

- Persona prompts that could be weaponized for prompt-injection or social engineering,
- Malicious `discuss-team.config.yaml` defaults,
- arxiv API abuse vectors,

please open a private security advisory via GitHub's "Security" tab on the repo, or email the maintainer.

We aim to acknowledge reports within 7 days and ship fixes within 30 days.
